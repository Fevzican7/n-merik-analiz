Fibonacci Dizisi

Antik çağlardan modern bilgisayar bilimlerine uzanan bu gizemli sayı dizisinin ardındaki algoritmik gücü ve matematiksel kusursuzluğu inceleyin.

Sınıf Algoritması Uygulamalı Deneyler

Leonardo of Pisa (Fibonacci)

Erken Dönem ve Hint-Arap Rakamları

Tam adıyla Leonardo Bonacci (1170 - 1250), İtalya'nın Pisa şehrinde doğmuştur. Babası Guglielmo, Kuzey Afrika'daki Bugia limanında bir tüccardı. Babasının yanına giderek orada eğitim alan genç Leonardo, Arap matematikçilerden günümüzde kullandığımız 10 tabanlı (0-9) Hint-Arap rakam sistemini öğrendi. Bu sistem, o dönem Avrupa'da hesaplama yapmayı imkansız kılan hantal Roma rakamlarına göre devrim niteliğindeydi.

Liber Abaci ve Modern Matematiğin Doğuşu

1202 yılında Pisa'ya dönen Fibonacci, bu sistemi detaylandırdığı başyapıtı Liber Abaci'yi yayımladı. Sıfır (0) rakamının basamak değeri olarak kullanılması, Avrupa'da modern matematik ve bilgisayar algoritmalarının temellerini attı.

Ünlü dizi ise bu kitapta yer alan "Tavşan Üreme Problemi" ile ortaya atıldı. "Her ay yeni bir çift doğuran tavşanların 1 yıl sonundaki sayısı ne olur?" sorusu, matematik literatürüne efsanevi bir diziyi kazandırdı.

Fibonacci Dizisi Genel Formülü:
Fn = Fn-1 + Fn-2

İlahi Oran: Altın Oran (Phi)

Altın Oran Nedir?

Altın Oran (Phi, 1.618...), matematikte ve sanatta, bir bütünün parçaları arasındaki en estetik ve en dengeli oran olarak kabul edilir. İki miktar arasındaki oran, bu iki miktarın toplamının büyük olanına oranına eşitse, bu oran "Altın Oran"dır.

Fibonacci dizisindeki ardışık iki sayının oranı, sayılar büyüdükçe kusursuz bir şekilde bu gizemli sabite yakınsar: 34/21 = 1.619, 55/34 = 1.617, 89/55 = 1.618...

Altın Oran (Phi) Değeri:
Φ =
1 + √5 2
 ≈ 1.618033...

Tarihte Nasıl Bulundu?

Tarihte ilk kez Antik Yunan'da ünlü matematikçi Öklid (M.Ö. 300) tarafından "Ekstrem ve Ortalama Oran" olarak tanımlanmıştır. Ancak mimarideki altın dikdörtgen oranları sebebiyle antik tapınakların (örneğin Partenon) inşasında da kullanıldığı gözlemlenmektedir.

Rönesans döneminde Luca Pacioli'nin yazdığı ve Leonardo da Vinci'nin resimlediği "De Divina Proportione" (İlahi Oran) adlı kitapla tüm sanat ve mimari dünyasına yayılmıştır. Günümüzde ise modern yazılım arayüzü tasarımlarının vazgeçilmezidir.

Akademik Dizi (Array) Yaklaşımı

Bellek üzerinde dizi (array) tahsis edilerek hesaplanan klasik yöntem. Hafıza kullanımı O(N), Zaman karmaşıklığı O(N)'dir.

Soru: İlk N Fibonacci terimini veren algoritmayı yazınız.
  1. BAŞLA
  2. OKU N // Kullanıcıdan terim sayısını al
  3. EĞER N < 1 VEYA N > 50 İSE
        YAZ "Hata: 1-50 arası girin" VE BİTİR
  4. int a[N]; // N boyutlu bellek (array) ayır
  5. a[0] = 0, a[1] = 1;
  6. Döngü (i = 2; i < N; i++)
    {
        a[i] = a[i-1] + a[i-2];
    }
  7. YAZ a Dizisi VE BİTİR

Sonuçlar dizilim blokları halinde burada belirecek...
FibonacciDizisi.java 
import java.util.Scanner;

public class FibonacciDizisi {
    public static void main(String[] args) {
        Scanner tarayici = new Scanner(System.in);
        System.out.print("Kaç terim istiyorsunuz? ");
        int n = tarayici.nextInt();

        // 1. Hata Kontrolü (Negatif veya 0 girilemez)
        if (n <= 0) {
            System.out.println("Lütfen pozitif bir sayı girin.");
            return;
        }

        // 2. Bellekte N elemanlı bir dizi (Array) ayırıyoruz
        // Hafıza Karmaşıklığı (Space Complexity): O(N)
        long[] dizi = new long[n];

        // 3. Dizinin ilk iki elemanı sabittir (0 ve 1)
        dizi[0] = 0;
        if (n > 1) {
            dizi[1] = 1;
        }

        // 4. Döngü ile kendinden önceki iki sayıyı topluyoruz
        // Zaman Karmaşıklığı (Time Complexity): O(N)
        for (int i = 2; i < n; i++) {
            dizi[i] = dizi[i - 1] + dizi[i - 2];
        }

        // 5. Sonuçları ekrana yazdırma
        System.out.println("Fibonacci Dizisi:");
        for (int i = 0; i < n; i++) {
            System.out.print(dizi[i] + " ");
        }
    }
}

Fibonacci Search (Arama Algoritması)

Klasik İkili Arama (Binary Search) diziyi sürekli ikiye bölme (/) işlemi yaparken, Fibonacci Arama sadece toplama (+) ve çıkarma (-) işlemleri yapar. Bilgisayar işlemcileri (CPU) için toplama ve çıkarma, bölme işleminden çok daha hızlıdır.

Soru: Sıralı bir dizide x sayısını arayan Fibonacci Algoritmasını yazınız.
  1. BAŞLA
  2. OKU arananDeger // Kullanıcıdan arananı al
  3. fibM2 = 0, fibM1 = 1, fibM = 1;
  4. Döngü (fibM < n) { fibM2=fibM1; fibM1=fibM; fibM=fibM2+fibM1; } // Sınırı bul
  5. baslangic = -1;
  6. Döngü (fibM > 1)
    {
        i = min(baslangic + fibM2, n - 1);
        EĞER dizi[i] < arananDeger İSE
            { fibM=fibM1; fibM1=fibM2; fibM2=fibM-fibM1; baslangic=i; }
        EĞER dizi[i] > arananDeger İSE
            { fibM=fibM2; fibM1=fibM1-fibM2; fibM2=fibM-fibM1; }
        DEĞİLSE { YAZ "Bulundu" VE BİTİR }
    }
  7. YAZ "Bulunamadı" VE BİTİR

FibonacciArama.java 
// Sıralı bir dizide hızlı arama yapan Fibonacci Algoritması
public static int fibonacciArama(int[] dizi, int arananDeger) {
    int n = dizi.length;

    // 1. Başlangıç Fibonacci sayılarını tanımlıyoruz
    int fibM2 = 0; // (m-2)'nci Fibonacci sayısı
    int fibM1 = 1; // (m-1)'nci Fibonacci sayısı
    int fibM = fibM2 + fibM1; // m'nci Fibonacci sayısı

    // 2. Dizinin boyutundan büyük veya eşit olan en küçük Fibonacci sayısını buluyoruz
    while (fibM < n) {
        fibM2 = fibM1;
        fibM1 = fibM;
        fibM = fibM2 + fibM1;
    }

    // Elenen kısımları takip etmek için başlangıç noktası (offset)
    int baslangic = -1;

    // 3. Arama Döngüsü Başlıyor
    while (fibM > 1) {
        // Geçerli bir indeks hesaplıyoruz (dizinin dışına çıkmamak için Math.min kullanıyoruz)
        int i = Math.min(baslangic + fibM2, n - 1);

        // DURUM 1: Eğer aranan değer sağ taraftaysa (Sol tarafı çöpe at)
        if (dizi[i] < arananDeger) {
            fibM = fibM1;       // Fibonacci değerlerini 1 adım geri al
            fibM1 = fibM2;
            fibM2 = fibM - fibM1;
            baslangic = i;      // Başlangıç noktasını sağa kaydır
        }
        // DURUM 2: Eğer aranan değer sol taraftaysa (Sağ tarafı çöpe at)
        else if (dizi[i] > arananDeger) {
            fibM = fibM2;       // Fibonacci değerlerini 2 adım geri al
            fibM1 = fibM1 - fibM2;
            fibM2 = fibM - fibM1;
        }
        // DURUM 3: Eşleşme bulundu!
        else {
            return i; // İndeksi geri döndür
        }
    }

    // Son elemanı kontrol ediyoruz (bazı durumlarda döngüde kaçabilir)
    if (fibM1 == 1 && baslangic + 1 < n && dizi[baslangic + 1] == arananDeger) {
        return baslangic + 1;
    }

    // Sayı dizide yoksa -1 döndür
    return -1;
}
📊 Performans Analizi

Arama Algoritmalarının Hız Karşılaştırması

1. Linear Search (Doğrusal Arama): Sayıları baştan sona tek tek kontrol eder. 1000 elemanlı dizide 1000 adım sürebilir. Çok yavaştır O(n).
2. Binary Search (İkili Arama): Diziyi sürekli tam ortadan ikiye böler (/). En az adımı atar ancak işlemciyi yoran bölme işlemi kullanır O(log₂ n).
3. Fibonacci Search: Diziyi Altın Oran'a göre böler. Sadece toplama (+) ve çıkarma (-) yaptığı için İkili Arama'ya göre birkaç adım fazla atsa da, donanım ve CPU seviyesinde çok daha performanslı çalışır O(log_φ n).

Doğrusal (Çok yavaş) İkili - Binary (Az adım, Bölme işlemi) Fibonacci (Optimum CPU)

Pascal Üçgeni ve Fibonacci Bağı

Blaise Pascal ve Tarihçesi

Blaise Pascal (1623-1662), Fransız matematikçi ve fizikçidir. Olasılık kuramının temellerini atan bilim insanlarından biridir.

Bu üçgenin özellikleri Pascal'dan yüzyıllar önce Hint, İranlı (Ömer Hayyam) ve Çinli matematikçiler tarafından biliniyordu. Ancak Pascal'ın bu konudaki "Traité du triangle arithmétique" adlı eseri o kadar kapsamlıydı ki, Batı dünyasında bu yapı onun adıyla anıldı.

Matematiğin En Büyük Sürprizi

Pascal üçgenindeki her sayı, üstündeki iki sayının toplamıdır (Kombinasyon mantığı). Ancak bu üçgenin içine gizlenmiş muazzam bir sır vardır: Fibonacci Dizisi!

Eğer Pascal üçgenindeki sığ köşegenleri (çapraz satırları) toplarsanız, kusursuz bir şekilde Fibonacci sayılarını elde edersiniz (1, 1, 2, 3, 5, 8, 13...). Evrenin farklı gibi görünen iki matematiksel konsepti, aynı potada birleşmektedir.

Soru: Pascal üçgeninin ilk N satırını çizen algoritmayı yazınız.
  1. BAŞLA
  2. OKU satirSayisi // Kullanıcıdan satır sayısını al
  3. EĞER satirSayisi < 1 VEYA satirSayisi > 15 İSE
        YAZ "Hata: 1-15 arası girin" VE BİTİR
  4. Döngü (i = 0; i < satirSayisi; i++)
    {
        Boşlukları Yazdır; // Piramit hizalaması için
        deger = 1;
        Döngü (j = 0; j <= i; j++)
        {
            YAZ deger;
            deger = deger * (i - j) / (j + 1); // Formül
        }
        YAZ YeniSatir;
    }
  5. BİTİR

Pascal Piramidi Çizici

Aşağıya çizilecek satır sayısını girin ve Pascal üçgeninin algoritmik inşasını canlı görün.

Pascal üçgeni burada görselleşecek...
PascalUcgeni.java 
import java.util.Scanner;

public class PascalUcgeni {
    public static void main(String[] args) {
        Scanner tarayici = new Scanner(System.in);
        System.out.print("Kaç satır çizilsin? ");
        int satirSayisi = tarayici.nextInt();

        // 1. Üçgeni satır satır çizmek için ana döngü başlar
        for (int i = 0; i < satirSayisi; i++) {

            // 2. Görsel Hizalama (Piramit şekli için boşluk bırakma)
            for (int bosluk = 0; bosluk < satirSayisi - i; bosluk++) {
                System.out.print("  ");
            }

            // 3. Her satırın en dıştaki ilk sayısı her zaman 1'dir
            int deger = 1;

            // 4. Satırdaki sayıları hesaplayıp yan yana yazdırıyoruz
            for (int j = 0; j <= i; j++) {
                // %4d, sayının 4 karakterlik alan kaplamasını sağlar
                System.out.printf("%4d", deger);

                // Kombinasyon Mantığı ile Sonraki Sayıyı Bulma:
                // Yeni Değer = Eski Değer * (Satır - Sütun) / (Sütun + 1)
                deger = deger * (i - j) / (j + 1);
            }

            // 5. Bir satır tamamen bittikten sonra alt satıra geçer
            System.out.println();
        }
    }
}

01

Kayıp Sayı Arama Oyunu

▼

Üretilen dizide eksik olan 5. sayıyı bulun. Doğru cevaplarsanız dizi otomatik yenilenir.

?, ?, ?, ?, ?

Algoritma Nasıl Çalışır?

Adım 1: Sistem, rastgele bir başlangıç noktası seçerek art arda 5 adet Fibonacci sayısı üretir.
Adım 2: Dizinin ilk 4 elemanını ekrana yazdırır ve 5. elemanı saklar.
Adım 3: Sizden gelen tahmini alır. Fibonacci kuralı gereği, dizinin 5. elemanı, her zaman 3. ve 4. elemanın toplamına eşit olmak zorundadır (Fn = Fn-1 + Fn-2).
Adım 4: Eğer girdiğiniz sayı bu toplama eşitse, başarılı sayılırsınız ve döngü yeni sayılarla baştan başlar.

02

Altın Oran Parçalayıcı

1.618 ile boyutları ayrıştır
▼

Bir uzunluk girin (örneğin bir çubuğun boyu), onu Altın Oran'a (1.618) göre en estetik iki parçaya bölelim.

-

Algoritma Nasıl Çalışır?

Mantık: Matematikte bir bütünü en estetik oranda bölmek için "Bütün / Büyük Parça = Büyük Parça / Küçük Parça" kuralı uygulanır ve bu daima 1.618033... (Phi) sayısını verir.
İşlem 1: Girdiğiniz toplam uzunluk doğrudan 1.6180339887 sayısına bölünür. Çıkan sonuç "Büyük Parça"nın uzunluğudur.
İşlem 2: Toplam uzunluktan, "Büyük Parça" çıkarılır. Kalan miktar "Küçük Parça"nın uzunluğudur.
Test: Son olarak algoritma sağlama yapar: Büyük Parçayı, Küçük parçaya böler. Sonuç 1.618 çıkıyorsa hesaplama kusursuzdur.

03

Dinamik Tavşan Üremesi

Tarihteki ilk Fibonacci problemi
▼

Fibonacci'nin ünlü sorusu: Her ay yeni bir çift doğuran tavşanların, belirttiğiniz ayın sonundaki toplam çift sayısı ne olur?

-

Algoritma Nasıl Çalışır (Dinamik Programlama)?

Bu problemi standart özyineleme (kendi kendini çağırma) ile çözmek bilgisayarı dondurabilir çünkü aynı aylar tekrar tekrar hesaplanır. Algoritma bunu önlemek için "Dinamik Programlama - Tablolama" kullanır.
Tablo Oluşturma: İstenilen ay sayısında boş bir liste (tablo) açılır. 1. ve 2. aylar 1 olarak doldurulur.
Döngüsel İlerleme: 3. aydan itibaren, her ayın tavşan sayısı, listeye kaydedilmiş bir önceki ay ile ondan önceki ayın sayılarının toplanıp yeni kutuya yazılmasıyla bulunur.
Sonuç: Hiçbir işlem ikinci kez yapılmaz. O(n) hızında anında sonuç üretilir.

04

Borsa Destek & Direnç Analizi

Fibonacci Retracement (Düzeltme) seviyeleri
▼

Bir hisse senedinin veya kripto paranın tepe ve dip fiyatını girin, olası düşüşlerde fiyatın nereden dönebileceğini (destek seviyelerini) bulalım.

-

Algoritma Nasıl Çalışır?

Finans piyasalarında kitle psikolojisi doğadaki oranlara (%38.2, %50, %61.8) uyma eğilimindedir. Yatırımcılar düzeltmelerin bu oranlarda biteceğine inanır.
1. Fark Hesaplama: Tepe fiyatından, dip fiyatı çıkarılarak "Dalga Boyu (Fark)" bulunur.
2. Oransal Çarpım: Bu fark, sırasıyla 0.236, 0.382, 0.500 ve 0.618 Altın Oran türevleriyle çarpılır.
3. Seviye Tespiti: Çıkan sonuçlar tepe noktasından tek tek çıkartılarak ekrana yansıtılır. Özellikle %61.8 (Altın Oran) piyasalarda en güçlü destek noktası olarak kabul edilir.

05

Kriptografi (Şifreleme)

Metni Fibonacci dizisiyle şifrele / çöz
▼

Girdiğiniz kelimenin her harfini, bulunduğu sıradaki Fibonacci sayısı kadar alfabede ileri kaydırarak şifreler.

-
-

Algoritma Nasıl Çalışır (Dinamik Kaydırma)?

Roma imparatoru Sezar şifrelemesinde her harf sabit bir sayı (örn: 3) kadar kaydırılırdı. Bu çok kolay çözülür. Bizim algoritmamız ise değişken şifre (dinamik anahtar) kullanır.
1. Anahtar Üretimi: Kelimenin uzunluğu kadar Fibonacci sayısı üretilir (Örn: "ALI" için 1, 2, 3 üretilir).
2. Şifreleme (İleri): 1. harf alfabede 1 adım ileri, 2. harf 2 adım ileri, 3. harf 3 adım ileri kaydırılır. Eğer harf 'Z'yi geçerse Modulo 26 matematiği ile başa 'A' harfine döner.
3. Deşifre (Geri): Çözülürken aynı işlem tersine yapılır. Karakterler ilgili sıradaki Fibonacci sayısı kadar geriye kaydırılır.

Sıkça Sorulan Sorular

1. Fibonacci dizisi tam olarak nedir?

İlk iki terimi 0 ve 1 olan, sonraki her terimin kendinden önceki iki terimin toplamıyla elde edildiği sayı dizisidir.

2. Nümerik Analiz derslerinde neden öğretilir?

Farklı algoritma tasarımlarının (Yinelemeli vs. Dinamik Döngüsel) işlemciyi ve belleği nasıl etkilediğini gösteren laboratuvar örneğidir.

3. Altın Oran ile Fibonacci dizisi nasıl ilişkilidir?

Ardışık iki Fibonacci sayısının oranı (F(n)/F(n-1)), n büyüdükçe Altın Oran'a (yaklaşık 1.6180339887) yakınsar. Bu yakınsama, dizinin en büyüleyici özelliklerinden biridir.

4. Fibonacci Search'ün avantajı nedir?

Doğrusal (Linear) arama her elemanı tek tek kontrol ederken, Fibonacci Search diziyi altın orana göre böler ve atlayarak ilerler. Büyük verilerde muazzam bir hız kazancı sağlar.

5. Doğada bu dizi nerelerde görülür?

Ayçiçeği çekirdeklerinin diziliminde, çam kozalaklarında, ananas kabuklarında, DNA sarmalında ve sarmal galaksilerin yapısında kusursuzca görülür.
